# 生成热更新资源
* 热更新框架： HybridCLR v6.1.0
* Unity版本： unity 2022.3.33 f1c1
* 首次创建/下拉工程，或者 HybridCLR/Unity 版本有更新，都需要在菜单栏通过 HybridCLR - Installer 打开的窗口内，执行一次 Install

* 策略：
	- AOT dlls: Assembly-Main
	- 热更新 dlls: Assembly-CSharp

* 主要路径（以git仓库目录为根目录）
	- 热更新 dll 的输出目录: ./Javrmn/HybridCLRData/HotUpdateDlls
	- 裁剪后的 AOT dll 输出目录: ./Javrmn/HybridCLRData/AssembliesPostIl2CppStrip
	- 生成的 link.xml 文件: ./Javrmn/Assets/HybridCLRGenerate/link.xml
	- 需要补充元数据的 AOT dlls: mscorlib, System, System.Core, Newtonsoft.Json


* 自动化打包：
	- 编辑器菜单: EditorTools - 打xx代理包
	- 参考脚本: Editor - MulAgent.cs
	- 大致实现: 
		+ 导出 Excel 数据
		+ 执行热更新框架的 GenerateAll 指令（相当于按顺序执行: 编译DLL - 生成 Il2CppDef - 生成 Link.xml - 生成裁剪后的 AOT dlls - 生成桥接函数、泛型引用等）
		+ 为编译生成的 AOT dlls 和 热更新 dlls 配置 AssetBundles 标签
		+ 构建 AssetBundles, 写入资源清单等文件
		+ 把新生成的资源，拷贝到待发布到目录: ./Data/Package/{打包的目标平台}/{最新热更新资源目录}
		+ 将记录了最新资源目录名的 hotUpdate.txt 更新 
	
* 将 {最新热更新资源目录} 与 hotUpdate.txt 上传至服务器对应目录
